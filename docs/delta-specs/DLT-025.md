# Delta Spec: DLT-025 - Display formatted markdown in REPL responses

## Story

As a developer, I want the REPL to render markdown formatting in agent responses so that headings, code blocks, lists, and emphasis are visually distinct instead of showing raw syntax.

## What It Does

Adds markdown-aware rendering to the terminal REPL channel using the `rich` library. Agent responses that contain markdown syntax (headings, bold, italic, code blocks, lists, links) are displayed with appropriate terminal formatting. Also migrates tool activity rendering from manual ANSI escape codes to unified styled output. Simplifies the prompt from `you>` to `>`.

## Requirements

| ID | Requirement | Status |
|----|-------------|--------|
| R0 | Render markdown-formatted agent responses with visual formatting in the terminal | Core goal |
| R1 | Render headings, bold, italic, code blocks (with syntax highlighting), lists, and links | Must-have |
| R2 | All terminal output uses a unified rendering approach — no manual ANSI escape codes | Must-have |
| R3 | Agent responses render incrementally as chunks arrive without waiting for the full response | Must-have |
| R4 | Preserve existing tool activity display behavior (styled status lines with tool-specific details) | Must-have |
| R5 | Simplify REPL prompt from `you>` to `>` | Must-have |
| R6 | Graceful fallback for terminals without color support | Out |

**Assumption:** TextChunk events correspond to complete SDK TextBlocks and arrive as coherent text — markdown syntax is not split across chunks. If this invariant is violated, individual chunks may render with formatting artifacts.

## Acceptance Criteria

**Markdown rendering (R0, R1, R3)**
- [ ] Given the agent responds with markdown headings, when rendered in the REPL, then headings display with visual emphasis (bold, color, or size distinction)
- [ ] Given the agent responds with `**bold**` or `*italic*` text, when rendered, then the text appears with appropriate terminal formatting
- [ ] Given the agent responds with a fenced code block, when rendered, then the code displays with syntax highlighting based on the language tag
- [ ] Given the agent responds with a markdown list, when rendered, then items display as a properly indented list
- [ ] Given the agent responds with a markdown link, when rendered, then the link text is displayed with the URL shown (e.g., underlined text or `[text](url)` with color)
- [ ] Given the agent responds with plain text (no markdown), when rendered, then it displays normally without artifacts
- [ ] Given multiple TextChunk events arrive in sequence, when each is rendered, then each chunk displays immediately with markdown formatting

**Unified rendering (R2)**
- [ ] Given the project dependencies, when inspected, then `rich` is listed in `pyproject.toml` dependencies
- [ ] Given the rendering module, when inspected, then no manual ANSI escape code strings are present

**Tool activity (R4)**
- [ ] Given the agent uses a known tool, when a ToolActivity event arrives, then a styled status line shows the tool name and key parameters (same information as before)
- [ ] Given the agent uses an unknown tool, when a ToolActivity event arrives, then the tool name is shown as a fallback

**Response boundaries (preserved)**
- [ ] Given the agent completes its response, when a Result event arrives, then the response is cleanly terminated and a new prompt appears

**Prompt (R5)**
- [ ] Given the REPL is running, when waiting for input, then the prompt displays `> ` instead of `you> `

**Error handling (preserved)**
- [ ] Given an error occurs, when an error event arrives, then the error message prints to stderr
