# Delta Spec: DLT-073 - Block Claude Code built-in cron tools in default config

## Story

As a system, I need the agent's default tool deny list to include Claude Code's built-in cron tools so that the persistent task system is never shadowed by ephemeral built-in alternatives.

## What It Does

Adds `CronCreate`, `CronDelete`, and `CronList` to the default `disallowed_tools` in `AgentSettings`. Every new Tachikoma installation will block these tools without manual configuration, preventing the agent from creating ephemeral in-memory cron jobs that silently vanish on exit.

## Requirements

| ID | Requirement | Status |
|----|-------------|--------|
| R0 | Agent blocks Claude Code built-in cron tools by default so they never shadow the persistent task system | Core goal |
| R1 | `CronCreate`, `CronDelete`, `CronList` included in `AgentSettings.disallowed_tools` default value | Must-have |
| R2 | Existing users with explicit `disallowed_tools` overrides in their config file are unaffected | Must-have |
| R3 | Auto-generated default config file reflects the updated deny list in its comments | Must-have |

## Acceptance Criteria

### Core blocking (R0, R1)

- [ ] Given a fresh `Settings()` with no config overrides, when `agent.disallowed_tools` is read, then it contains `["AskUserQuestion", "CronCreate", "CronDelete", "CronList"]`
- [ ] Given the coordinator is initialized with the default disallowed tools, when it builds `ClaudeAgentOptions`, then the options include all four tools in the deny list

### Existing config preservation (R2)

- [ ] Given a config file with an explicit `disallowed_tools = ["AskUserQuestion"]`, when loaded, then only `["AskUserQuestion"]` is in the deny list (user's explicit value takes precedence over defaults)

### Generated config comments (R3)

- [ ] Given no config file exists, when `_generate_default_config` runs, then the generated file contains `disallowed_tools` with `"AskUserQuestion"`, `"CronCreate"`, `"CronDelete"`, and `"CronList"` in the commented-out default value
