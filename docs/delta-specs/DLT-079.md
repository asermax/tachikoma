# Delta Spec: DLT-079 - Escape markdown-sensitive characters in Telegram output

## Story

As a user, I want tool activity output in Telegram to render correctly when it contains special characters (asterisks, underscores, backticks) so that I can read tool status lines and summaries without broken formatting.

## What It Does

Wraps dynamic tool arguments (file paths, search patterns, shell commands) in inline code formatting within Telegram tool activity lines and summaries. This prevents markdown-sensitive characters in glob patterns (`**/*.ts`), regex (`git.*push`), dunder filenames (`__init__.py`), and shell commands from being misinterpreted as markdown formatting by the `telegramify_markdown` library. Single quotes around patterns in summaries are dropped since inline code already provides visual grouping.

## Requirements

| ID | Requirement | Status |
|----|-------------|--------|
| R0 | Tool activity output renders correctly in Telegram regardless of special characters in patterns, paths, and commands | Core goal |
| R1 | Dynamic tool arguments are wrapped in inline code (backticks) to prevent markdown interpretation and improve readability | Must-have |
| R2 | Inline code applies to Grep, Glob, Bash, Read, Edit, Write arguments — Agent and ToolSearch are excluded (controlled content) | Must-have |
| R3 | Both live tool lines (TOOL_DISPLAY) and transition summaries (TOOL_SUMMARY) are covered | Must-have |
| R4 | Changes are Telegram-specific — display.py (shared by REPL) remains unchanged | Must-have |
| R5 | Handle backtick collision in Bash commands by using double-backtick wrapping | Must-have |
| R6 | Drop single quotes around Grep/Glob patterns in summaries — inline code provides visual grouping | Must-have |
| R7 | Status messages and error messages are unaffected | Out |

## Acceptance Criteria

### Tool argument formatting (R1, R2, R3)

- [ ] Given a Grep tool activity with pattern `git.*push`, when the live tool line is rendered in Telegram, then the pattern appears in inline code: `🔧 Searching for `git.*push``
- [ ] Given a Glob tool activity with pattern `**/*.ts`, when the live tool line is rendered in Telegram, then the pattern appears in inline code: `🔧 Globbing `**/*.ts``
- [ ] Given a Read tool activity with file path `/src/tachikoma/__init__.py`, when the live tool line is rendered in Telegram, then the path appears in inline code: `🔧 Reading `/src/tachikoma/__init__.py``
- [ ] Given an Edit tool activity with file path `test_utils.py`, when the live tool line is rendered in Telegram, then the path appears in inline code: `🔧 Editing `test_utils.py``
- [ ] Given a Write tool activity with file path `output.json`, when the live tool line is rendered in Telegram, then the path appears in inline code: `🔧 Writing `output.json``
- [ ] Given a Bash tool activity with command `ls -la *.py`, when the live tool line is rendered in Telegram, then the command appears in inline code: `🔧 Running: `ls -la *.py``
- [ ] Given a Bash tool activity with a description field, when the live tool line is rendered in Telegram, then the description appears in inline code: `🔧 Running: `install dependencies``
- [ ] Given a tool→text transition with Grep activities, when the summary is rendered, then pattern arguments appear in inline code within the summary text
- [ ] Given a tool→text transition with multiple tool types, when the summary is rendered, then each tool's arguments appear in inline code within the combined summary
- [ ] Given an Agent tool activity, when the live tool line is rendered, then the description is NOT wrapped in inline code (excluded tool)
- [ ] Given a ToolSearch tool activity, when the live tool line is rendered, then the query is NOT wrapped in inline code (excluded tool)

### Backtick collision (R5)

- [ ] Given a Bash tool activity with command containing backticks (e.g., `` echo `date` ``), when the live tool line is rendered, then double-backtick wrapping is used so the command displays correctly
- [ ] Given a Bash tool activity with a description (no backticks), when rendered, then single-backtick wrapping is used (normal case)

### Quote removal (R6)

- [ ] Given a Grep tool summary, when rendered, then the pattern has no surrounding single quotes — inline code provides the visual grouping (e.g., `searching for `pattern`` not `searching for '`pattern`'`)
- [ ] Given a Glob tool summary, when rendered, then the pattern has no surrounding single quotes

### Telegram-specific isolation (R4)

- [ ] Given the REPL channel renders tool activity, when displaying a Grep with pattern `git.*push`, then the output is unchanged from current behavior (no inline code wrapping)
- [ ] Given display.py's TOOL_DISPLAY and TOOL_SUMMARY formatters, when inspected, then they remain unchanged — formatting is applied at the Telegram rendering layer
