## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Bootstrap hook: create `projects/` directory, discover submodules from `.gitmodules`, initialize and pull all in parallel with 1 retry, check out default branch per submodule | |
| **S2** | Projects post-processor: iterate workspace submodules, check each for uncommitted changes, spawn Haiku agent per dirty submodule (in parallel) to commit, then push each; runs in finalize phase before GitProcessor | ⚠️ |
| **S3** | `register_project` MCP tool: runs `git submodule add <url> projects/<name>`, checks out default branch, exposed to coordinator during conversations | ⚠️ |
| **S4** | `deregister_project` MCP tool: checks for uncommitted changes (warns + requires force flag if present), runs `git submodule deinit`, removes from `.gitmodules`, removes directory | |
| **S5** | Projects context provider: pre-processing provider that reads `projects/` submodules and returns XML-tagged context block with project names, paths, and current branches (or commit hash if detached) | |

### Flagged Unknowns

#### S2 — Finalize phase ordering
The current post-processing pipeline runs processors within a phase in parallel. Both the projects processor and the git processor belong in "finalize", but the projects processor MUST complete before GitProcessor starts (so GitProcessor commits the updated submodule refs). Options include: (a) adding a new phase between main and finalize, (b) adding ordering/priority within finalize, (c) sequential execution in finalize. Needs investigation during design.

#### S3 — MCP tool exposure to coordinator
Currently, MCP tools are only used in forked post-processing sessions (e.g., pending signals tools in context processor). The register/deregister project tools need to be available to the main coordinator agent during live conversations. The mechanism for passing MCP servers to the coordinator's SDK client needs investigation — likely via `ClaudeAgentOptions.mcp_servers` at coordinator creation, but this pattern doesn't exist in the codebase yet.
