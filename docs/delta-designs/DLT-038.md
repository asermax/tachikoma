### Flagged Unknowns

#### S2 — Staleness detection via directory mtime
The skills directory mtime only reflects direct children changes (new/deleted subdirectories), not modifications to files within subdirectories (e.g., editing `skill-name/SKILL.md`). Need to determine: should we check the mtime of the top-level `skills/` directory only (catches additions/deletions but not in-place edits), walk subdirectories to check all mtimes (reliable but more I/O), or use a simpler approach like always re-scanning (trivial cost for < 20 skills)? Note: choosing "always re-scan" would effectively make R2 a no-op — this is an acceptable outcome given negligible cost, but would mean the R2 ACs are satisfied vacuously. The design phase should resolve this tradeoff.
