## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Remove tolerance window from cron evaluation — change `should_fire` to only check `next_fire <= now_tz`; set both `scheduled_for` and `last_fired_at` to the actual cron match time (`next_fire` converted to UTC) instead of `now_utc` | |
| **S2** | Modify `get_active_instance_for_definition` to accept a `scheduled_for` parameter and match instances where `scheduled_for` equals the current cron match time — covers pending, running, and completed states (excludes failed to allow retry) | |
| **S3** | Add debug/info log when the safety net suppresses a duplicate instance | |
