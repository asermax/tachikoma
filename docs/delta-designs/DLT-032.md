### Flagged Unknowns

#### S2 — Built-in skill deployment
How should a built-in skill be made available in the workspace? The design should favor the bootstrap hook pattern (consistent with existing `skills_hook`, `memory_hook`, etc.) and must be discoverable by the existing skill registry without modifications to registry scanning logic. Key design questions:
- Should the hook overwrite on every launch (ensuring latest content), skip if exists (preserving user modifications), or version-check (update only when bundled version is newer)?
- Where does the source content live in the application package?
