## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Change `TaskSettings.timezone` default from `""` to resolved system timezone (via `ZoneInfo("localtime")` or similar), with Pydantic validation rejecting invalid timezone strings at startup | |
| **S2** | Refactor `scheduler.py:_get_timezone()` to use `settings.timezone` directly (no more empty-string fallback logic) | |
| **S3** | Add `timezone: ZoneInfo` parameter to `create_task_tools_server()` and plumb it from `__main__.py` using the resolved `TaskSettings.timezone` | |
| **S4** | Update `_parse_schedule()` to accept a `tz` parameter — naive datetimes get `tz` instead of UTC; datetimes with explicit offset are preserved as-is | |
| **S5** | Update `_format_schedule()` to accept a `tz` parameter and convert one-shot datetimes to that timezone for display | |
| **S6** | Update `create_task` and `update_task` tool description strings to document timezone behavior and accepted formats | |
| **S7** | Update the `SYSTEM_PREAMBLE` Tasks/Scheduling section to document that bare datetimes are interpreted as local time | |
