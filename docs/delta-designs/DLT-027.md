## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Session data model — dataclass with fields: id (UUID), sdk_session_id, transcript_path, started_at, ended_at. Computed `status` property derives open/closed/interrupted from field presence (ended_at + sdk_session_id). No persisted status column. | |
| **S2** | SQLite repository — create/read/update sessions; DB at `settings.workspace.data_path / "sessions.db"`; handles schema creation on first access; queries by ID and time range | |
| **S3** | Session registry — dedicated component exposing create_session(), close_session(), recover_interrupted(); called by coordinator and boundary detectors | |
| **S4** | Bootstrap hook — registered via DLT-023's hook system; on every launch, calls `registry.recover_interrupted()` to detect open sessions and mark them as interrupted with best-effort end timestamps; idempotent | |
