## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | **Git bootstrap hook** — Registered with DLT-023 after workspace hook. Checks for `.git` in workspace, runs `git init` if missing, configures repo-local `user.name`/`user.email` with fixed identity, creates initial empty commit. No `.gitignore` created. Idempotent. | |
| **S2** | **Phased pipeline execution** — Extends DLT-008's pipeline to support phases. Processors declare a phase (`main` or `finalize` — fixed enum). Pipeline runs main-phase processors in parallel first, then finalize-phase processors in parallel after all main processors complete. Error isolation applies across phases (finalize runs even if main processors fail). | |
| **S3** | **Git post-processor** — Registered in the `finalize` phase. Checks for uncommitted changes via subprocess `git status` (modified tracked + untracked files). If clean, returns no-op. If dirty, spawns the Haiku agent via fresh `query()` with resource limits. After agent completes, checks workspace status and logs warning if changes remain. | |
| **S4** | **Haiku commit agent prompt** — Instructs the agent to: inspect `git status` and `git diff`, group changes by workspace subdirectory/purpose, create one `git add` + `git commit` per group with a descriptive message. Prompt explicitly restricts to safe git commands only (no push, branch, reset, checkout). Agent restrictions are prompt-level, consistent with DLT-008 memory processors. | ⚠️ |

### Flagged Unknowns

#### S4 — Prompt design for cohesive grouping
The prompt is the core mechanism that determines whether the agent correctly groups changes and produces quality commit messages. The grouping dimensions (by subdirectory/purpose) are defined in the spec, but the prompt wording that reliably achieves this behavior with Haiku needs iteration. Key questions: How detailed should grouping instructions be? Should the prompt provide explicit directory-to-group mappings, or let the agent infer grouping from directory structure? How does the agent handle large diffs that may approach context limits?
