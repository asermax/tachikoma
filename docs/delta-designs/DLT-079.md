## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Inline code wrapping utility in `telegram.py` — wraps text in single backticks, or double backticks (with space padding) when text contains backticks | |
| **S2** | Telegram-specific tool formatting in `telegram.py` — applies code wrapping to arguments for Grep, Glob, Bash, Read, Edit, Write in both `handle_tool` (live lines) and `handle_text`/`finalize` (summaries) | ⚠️ |

### Flagged Unknowns

#### S2 — Summary path integration
The live tool line (`handle_tool`) is straightforward: format directly from `activity.tool_input` with code-wrapped arguments. But the summary path is trickier: `summarize_tool_activity()` in `display.py` composes a finished string using `TOOL_SUMMARY` formatters, so the arguments are already baked into the output. The design phase needs to decide how to get code-wrapped arguments into summaries without modifying `display.py` — likely a Telegram-specific summary builder or a parallel set of summary formatters in telegram.py that `summarize_tool_activity` can accept.
