## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Tool activity collector — accumulate `ToolActivity` objects per tool→text boundary in the renderer, building a list that gets reset after each summary is generated | |
| **S2** | Summary generator — rule-based function that takes a list of `ToolActivity` objects and produces a single-line human-readable description. Reuses formatting patterns from the existing `TOOL_DISPLAY` map for per-tool detail extraction. Groups by tool type, uses adaptive aggregation (>2 of same type → count, otherwise lists details), caps at 5 groups. Falls back to tool name for unknown tools and to default placeholders for malformed input | |
| **S3** | Telegram renderer integration — replace the hardcoded "🔧 Ran tools" string in `handle_text()` with a call to the summary generator, passing the collected activities | |
| **S4** | Channel-agnostic module placement — summary generator lives in `display.py` alongside `TOOL_DISPLAY`, making it available to any channel | |

### Flagged Unknowns

None — all mechanisms are well-understood.
