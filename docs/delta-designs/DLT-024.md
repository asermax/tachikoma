## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | Enable packaging in pyproject.toml: set `package = true`, add `[build-system]` with `uv_build`, add `[project.scripts]` entry pointing to cyclopts app, add PyPI metadata (description, readme, license, classifiers, URLs) | |
| **S2** | Restructure CLI: convert `@app.default` to `@app.command` for `run` subcommand, set `run` as the default command in cyclopts so bare `tachikoma` invokes it | |
| **S3** | Package data inclusion: configure build to include non-Python files (`.md` files in builtin skills directories) in the distribution | |
| **S4** | Add MIT LICENSE file at project root | |
| **S5** | Add python-semantic-release configuration to pyproject.toml: `version_toml`, conventional commit parser, `build_command` (updates uv.lock + builds), changelog settings | |
| **S6** | Create GitHub Actions CD workflow (`.github/workflows/release.yml`): triggered on push to master, runs quality gates, runs python-semantic-release for version determination, publishes to PyPI via trusted publisher if a release was created; includes concurrency control | |
| **S7** | Update justfile recipes: remove `PYTHONPATH=src` (no longer needed once package is installed in dev env via `uv sync`), ensure dev workflow preserved | |
| **S8** | Add installation section to README.md covering: install via `uv tool install`, first-run config generation, `ANTHROPIC_API_KEY` setup, channel selection, and upgrade path | |

### Flagged Unknowns

None — all mechanisms are understood.
