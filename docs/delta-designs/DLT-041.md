## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | `SessionContextEntry` domain dataclass + `SessionContextRecord` ORM model (1 entry per owner, text content) | |
| **S2** | `SessionContextRepository` with save and load-by-session methods | |
| **S3** | Context assembly function: loads entries from DB, builds full system prompt append string (prepends SYSTEM_PREAMBLE, wraps entries in XML tags by owner, follows fixed ordering convention) | |
| **S4** | Coordinator integration: saves entries at each lifecycle point (session creation → foundational; post-pipeline → provider results; boundary detection → transition context); uses assembly function in `_build_options()` on every client creation | |
| **S5** | Pre-processing injection path change: `assemble_context()` superseded — results go to system prompt via S4 persistence + S3 assembly | |
| **S6** | Fork helper generalization: optional context restoration parameter that uses S3 to build system prompt append for the fork's `ClaudeAgentOptions` | |
| **S7** | Database migration: new `session_context` table with FK to sessions | |

### Flagged Unknowns

None — all mechanisms are well-understood patterns from the existing codebase.
