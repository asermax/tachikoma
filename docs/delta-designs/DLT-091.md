### Flagged Unknowns

#### S1 — SQLite column drop approach

The existing codebase uses pragma-based column checks that only *add* columns — dropping a column is unprecedented. SQLite supports `ALTER TABLE ... DROP COLUMN` since version 3.35.0 (2021), but has restrictions (no primary key, no indexed columns, no columns referenced by other tables). The `notify` column should qualify for direct drop, but the design phase needs to verify this and decide whether to use `ALTER TABLE DROP COLUMN` or a table-rebuild approach.
