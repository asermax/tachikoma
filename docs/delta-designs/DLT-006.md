| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | **ContextProvider ABC** — Abstract base class with `async provide(message: str) -> ContextResult \| None`. Each provider receives the user message, returns a named context result or None. Domain-agnostic, no SDK coupling. | |
| **S2** | **ContextResult dataclass** — `tag: str, content: str`. Represents a named XML-tagged context block (e.g., tag=`"memories"`, content=`"Relevant files: ..."`). | |
| **S3** | **PreProcessingPipeline** — Manages registered `ContextProvider`s. `register(provider)` to add. `run(message) -> list[ContextResult]` runs all providers in parallel via `asyncio.gather(return_exceptions=True)`, collects successful results, logs failures. | |
| **S4** | **Context assembly utility** — Assembles `list[ContextResult]` into an XML-tagged string (each result wrapped in `<{tag}>...</{tag}>`). Used by the coordinator to format the enriched message. | |
| **S5** | **Memory context provider** — `MemoryContextProvider(ContextProvider)` that makes a standalone `query()` call with a Haiku model and tools (Read, Glob, Grep, Agent for Explore subagent), `cwd` set to workspace path. Prompt instructs the agent to use Explore tools to search `memories/` and return a ranked list of the most relevant files (up to 10) with a short summary for each, plus instructions for the main agent to read further if necessary. Returns `ContextResult(tag="memories", content=...)` or None if no relevant files found. | ⚠️ |
| **S6** | **Coordinator integration** — In `send_message()`, after session creation and before `client.query()`: run `pipeline.run(message)`, assemble results into XML blocks via S4, prepend to user message text. Pipeline is an optional constructor dependency (if None, skip pre-processing). | |

### Flagged Unknowns

#### S5 — Memory search prompt design
How should the exploration prompt be structured for effective memory retrieval? Key questions: How to instruct the agent to balance thoroughness (search all three memory types: episodic, facts, preferences) with relevance (don't return irrelevant content)? What format should the ranked file list follow? How to ensure the agent produces consistent, parseable output? Additionally, giving the Haiku agent the Agent tool (for Explore subagent) introduces recursive sub-agent calls — the prompt should constrain the agent's exploration scope to keep cost and latency reasonable.
