| **S1** | **Session summary field** — Extend the session model (`Session` dataclass + `SessionRecord` ORM) with a `summary: str \| None` field. Add an `update_summary()` method to the registry. | |
| **S2** | **Summary post-processor** — A `PostProcessor` implementation that forks the SDK session, prompts the agent to produce a concise, bounded rolling summary of the conversation so far, and stores it via `registry.update_summary()`. | ⚠️ |
| **S3** | **Per-message pipeline instance and trigger** — A second `PostProcessingPipeline` instance (separate from the session pipeline) registered with the summary processor. The coordinator triggers it after each agent response completes, launching it as a background `asyncio.Task`. | |
| **S4** | **Pending task lifecycle** — The coordinator stores a reference to the background per-message task. Before boundary detection, it awaits completion (logging errors, not propagating). On graceful shutdown, it awaits any pending task before exiting. | |
| **S5** | **Boundary detector** — A callable that takes the incoming message text and the current session summary, makes an independent SDK `query()` call (separate from the active conversation), and returns a verdict (continuation or topic shift). | ⚠️ |
| **S6** | **Boundary check integration** — Intercept the coordinator's `send_message()` flow: after awaiting pending post-processing, read the current session summary, run the boundary detector (skip if no session or no summary), and branch based on the verdict. | |
| **S7** | **Session transition orchestrator** — On topic shift: close current session in registry, capture session + summary, fire async session post-processing (S9), reset SDK conversation context (S8), create new session, rebuild system prompt with previous summary, then proceed with message processing. On partial failure mid-sequence, ensure the system reaches a usable state (new Tachikoma session created even if SDK reset fails). | ⚠️ |
| **S8** | **SDK conversation context reset** — Disconnect the existing `ClaudeSDKClient` and reconnect (or construct a new instance) to start a fresh SDK conversation without prior context. Must support rebuilding `ClaudeAgentOptions` with the updated system prompt (previous conversation summary). | ⚠️ |
| **S9** | **Async session post-processing** — Fire the existing session-level `PostProcessingPipeline` as a background `asyncio.Task` when a session closes via boundary detection. Track all background tasks; on graceful shutdown, await remaining tasks. Log errors from background tasks. | |

### Flagged Unknowns

#### S2 — Summary prompt design
How should the summary prompt be structured for effective rolling summaries? Key questions: Should it see the full transcript via session fork, or receive the previous summary + latest exchange? What's the target summary length/bound? How to handle very long conversations where the summary might grow stale or too long?

#### S5 — Boundary detection prompt design
How to structure the classification prompt for reliable boundary detection? The prompt needs to produce a clean binary signal (continuation/shift) from the summary + new message. How to handle ambiguous cases (e.g., related but distinct topics, gradual topic drift)? What's the response format — structured JSON, a single word, or something else?

#### S7 — Session transition ordering and failure recovery
The session transition involves multiple steps with ordering dependencies: close session → fire async post-processing → reset SDK → create new session → inject summary → process message. What happens if any step fails mid-sequence? How to ensure the system ends up in a consistent, usable state? Does the system prompt rebuild require a new `ClaudeSDKClient` instance (since `SystemPromptPreset` is set in `ClaudeAgentOptions` at construction time)?

#### S8 — SDK client reconnection
Does `ClaudeSDKClient` support disconnect followed by reconnect on the same instance? Or does a new client need to be constructed? If a new client is needed, how does this interact with the `SystemPromptPreset` (which is set in `ClaudeAgentOptions` at construction)? This needs investigation of the SDK API.
