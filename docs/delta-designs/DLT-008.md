## Shape

| Part | Mechanism | Flag |
|------|-----------|:----:|
| **S1** | **Post-processing pipeline** — Accepts registered processors, runs them all in parallel via `asyncio.gather(return_exceptions=True)`. Takes a closed `Session` object as input. Logs individual processor failures without failing the pipeline. Serializes concurrent pipeline invocations. | |
| **S2** | **Pipeline integration with coordinator** — Coordinator receives the pipeline as an optional dependency. On session close, if the closing session has a valid `sdk_session_id`, awaits the pipeline. Pipeline uses the standalone `query()` function (independent of `ClaudeSDKClient`), so it can run after the coordinator begins its shutdown sequence. | |
| **S3** | **Memory processor base** — Common interface for memory processors. Each forks the session via the standalone `query(resume=sdk_session_id, fork_session=True)` with `ClaudeAgentOptions` granting full workspace access (no max_turns, no max_budget_usd). The processor code's only job is to fork the session with the right prompt and consume the async iterator — the forked agent (LLM) does all file reading/writing/deleting autonomously. | |
| **S4** | **Episodic memory processor** — Forks session with a prompt directing the agent to read `memories/episodic/`, analyze the conversation, and create or update date-stamped summary files (`YYYY-MM-DD.md`). The agent decides what to write; creating nothing is valid. | |
| **S5** | **Facts memory processor** — Forks session with a prompt directing the agent to read `memories/facts/`, identify new factual information, and create, update, or delete topic-named files. The agent decides what changed; creating nothing is valid. | |
| **S6** | **Preferences memory processor** — Forks session with a prompt directing the agent to read `memories/preferences/`, identify expressed preferences, and create, update, or delete topic-named files. The agent decides what changed; creating nothing is valid. | |
| **S7** | **Memory bootstrap hook** — Creates `memories/` and its three subdirectories on first run. Registered after workspace hook in `__main__.py`. Follows subsystem-owned hook pattern from DLT-005. | |
