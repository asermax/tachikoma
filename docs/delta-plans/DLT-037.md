**Known edge case:** If the sequence is Status("Thinking...") → Error (no text), `_current_message_id` is set by the status message. The error would be sent silently and `notify()` would copy the status message ("_Thinking..._"). This is acceptable — the user still receives a push notification, and it's a very rare edge case where the agent fails after sending a status but before any text.

**Constructor default:** `ResponseRenderer` defaults `push_notifications=False` (diverging from config default of `True`) for backward compatibility with existing tests that don't pass the flag. This is intentional.

**Test convention note:** Existing telegram tests use `unittest.mock` directly rather than `mocker` fixture (DES-001 preference). New tests follow existing file convention for consistency.
