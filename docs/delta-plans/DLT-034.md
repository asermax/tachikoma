#### Step 1: Add TOOL_SUMMARY map and summarize_tool_activity() to display.py

- Modify: `src/tachikoma/display.py`
- What to do:
  1. Import `os.path.basename` (for file path extraction in Read/Edit/Write formatters)
  2. Import `ToolActivity` from `tachikoma.events`
  3. Add a `TOOL_SUMMARY` dict mapping tool names to callables `(dict) -> str` that return lowercase verb-phrase fragments. Each formatter falls back gracefully on missing keys:

  ```python
  # In src/tachikoma/display.py, after TOOL_DISPLAY:

  # Per-tool summary formatters — each returns a lowercase verb-phrase fragment
  # Falls back to generic placeholder when tool_input is missing expected keys
  TOOL_SUMMARY: dict[str, Callable[[dict[str, Any]], str]] = {
      "Read": ...,      # "read {basename}" | "read a file"
      "Grep": ...,      # "searched for '{pattern}'" | "searched for a pattern"
      "Glob": ...,      # "globbed '{pattern}'" | "globbed a pattern"
      "Bash": ...,      # description (lowercased first char) | truncated command | "ran a command"
      "Edit": ...,      # "edited {basename}" | "edited a file"
      "Write": ...,     # "wrote {basename}" | "wrote a file"
      "ToolSearch": ..., # "searched tools"
  }
  ```

  4. Add aggregated form definitions — a dict mapping tool names to callables `(int) -> str`:

  ```python
  # Aggregated phrasing for tools with count > 2
  _TOOL_AGGREGATE: dict[str, Callable[[int], str]] = {
      "Read": ...,       # "read {count} files"
      "Grep": ...,       # "ran {count} searches"
      "Glob": ...,       # "ran {count} glob searches"
      "Bash": ...,       # "ran {count} commands"
      "Edit": ...,       # "edited {count} files"
      "Write": ...,      # "wrote {count} files"
      "ToolSearch": ..., # "ran {count} tool searches"
  }
  ```

  5. Add `summarize_tool_activity(activities: list[ToolActivity]) -> str` implementing the algorithm from design S3:
     - Return `""` if list is empty
     - Group activities by `tool_name` preserving first-seen order (dict insertion order)
     - For each group: if count > 2, use `_TOOL_AGGREGATE` (falling back to `"used {tool_name} {count} times"`); if count ≤ 2, format each individually via `TOOL_SUMMARY` (falling back to `"used {tool_name}"`)
     - If more than 5 tool-type groups, keep first 5 and append `"and more"` as a trailing element
     - Flatten all phrases into a single ordered list
     - Join: 1 item → as-is; 2 items → `"A and B"`; 3+ items → `"A, B, and C"` (Oxford comma)
     - Capitalize the first character of the joined string
     - Return

- Test: Unit tests in Step 2

#### Step 3: Replace flag-based tool tracking with activity list in ResponseRenderer

- Modify: `src/tachikoma/telegram.py`
- What to do:
  1. Import `summarize_tool_activity` from `tachikoma.display`
  2. In `__init__()`: remove `_had_tools` and `_tools_marker_inserted`, add `_tool_activities: list[ToolActivity] = []`
  3. In `reset()`: remove `_had_tools`/`_tools_marker_inserted` resets, add `self._tool_activities = []`
  4. In `handle_tool()`: append activity to `_tool_activities` list. Keep existing tool line display logic unchanged. Remove `_had_tools = True` and `_tools_marker_inserted = False`.
  5. In `handle_text()`: replace `_had_tools and not _tools_marker_inserted` check with `self._tool_activities` emptiness check. When non-empty: generate summary, insert marker, **clear `_tool_activities`**, clear `_tool_line`:

  ```python
  # In handle_text(), the tool→text transition block becomes:

  if self._tool_activities:
      if self._buffer and not self._buffer.endswith("\n"):
          self._buffer += "\n"

      summary = summarize_tool_activity(self._tool_activities)
      self._buffer += f"_🔧 {summary}_\n"
      self._tool_activities = []  # Clear — each transition gets independent summary
      self._tool_line = None
  ```

  6. In `finalize()`: add `self._tool_activities = []` — **new behavior** (current `finalize()` only clears `_tool_line`). Needed for the shutdown path where `_on_shutdown` calls `finalize()` without a subsequent `reset()`.
