#### Step 2: Create PreProcessingPipeline class

- Modify: `src/tachikoma/pre_processing.py`
- What to do:
  - Add `PreProcessingPipeline` class with:
    - `__init__`: initializes `_providers: list[ContextProvider] = []`
    - `register(provider: ContextProvider) -> None`: appends provider to list
    - `async run(message: str) -> list[ContextResult]`:
      - If no providers, return empty list immediately
      - Run all providers via `asyncio.gather(*[p.provide(message) for p in self._providers], return_exceptions=True)`
      - Iterate results with `zip(self._providers, results, strict=True)`
      - Log failures per DES-002 using `_log.exception()` with provider class name and error
      - Collect successful non-None ContextResult instances
      - Return the collected list
  - **No lock** — design doc explicitly states this (session registry prevents concurrent first-messages)
  - Mirror the error-logging pattern from `PostProcessingPipeline.run()` in `post_processing.py`

#### Step 5: Create MemoryContextProvider

- Create: `src/tachikoma/memory/context_provider.py`
- What to do:
  - Add module docstring explaining the memory context provider's purpose
  - Import `Path` from `pathlib`, `query` and `ClaudeAgentOptions` from `claude_agent_sdk`, `ResultMessage` from `claude_agent_sdk.types`, `loguru.logger`
  - Import `ContextProvider`, `ContextResult` from `tachikoma.pre_processing`
  - Create bound logger: `_log = logger.bind(component="memory_context")`
  - Define `MEMORY_SEARCH_PROMPT` constant — the prompt instructs the agent to:
    - Search `memories/episodic/`, `memories/facts/`, `memories/preferences/` directories
    - Use Glob to discover files, Grep to narrow by keywords/topics from the user message, Read to verify relevance
    - Return a ranked markdown list of up to 10 most relevant files with path and short summary
    - Include instructions for the main agent to read referenced files for more detail
    - If no relevant memories found (including when directories are empty), respond with exactly `NO_RELEVANT_MEMORIES`
    - The user's message is embedded into the prompt so the agent knows what to search for
  - Define `MemoryContextProvider(ContextProvider)`:
    - `__init__(self, cwd: Path)`: stores `self._cwd = cwd`
    - `async def provide(self, message: str) -> ContextResult | None`:
      - Build the prompt by embedding the user message into MEMORY_SEARCH_PROMPT
      - Create `ClaudeAgentOptions(model="opus", effort="low", max_turns=8, allowed_tools=["Read", "Glob", "Grep"], permission_mode="bypassPermissions", cwd=self._cwd)`
      - Iterate `async for sdk_message in query(prompt=..., options=...):`
      - If `isinstance(sdk_message, ResultMessage)`:
        - If `sdk_message.is_error` → log warning, return None
        - If `sdk_message.result` is not None and doesn't contain `NO_RELEVANT_MEMORIES` → return `ContextResult(tag="memories", content=sdk_message.result)`
      - If loop completes without a valid result → return None

```python
# Embeds {message} placeholder for the user's actual message

# - Uses standalone query() (same pattern as git/processor.py query_and_consume)
# - Extracts result from ResultMessage.result (design decision: ResultMessage.result for output extraction)
# - Returns ContextResult(tag="memories") or None
```

#### Step 9: Integrate pre-processing in send_message()

- Modify: `src/tachikoma/coordinator.py`
- What to do:

  - Initialize `is_new_session = False` before the registry block
  - Inside the existing `if active is None:` branch, set `is_new_session = True` **immediately after** `active = await self._registry.create_session()` succeeds. If `create_session()` raises, the except catches it and `is_new_session` stays `False` — pre-processing is correctly skipped
  - After the existing try/except session block and before `await self._client.query(text)`:

    - On exception: `_log.exception("Pre-processing failed: err={err}", err=str(exc))` — original `text` used as fallback since the reassignment didn't happen
  - The `text` variable (enriched or original) is then passed to `self._client.query(text)`
