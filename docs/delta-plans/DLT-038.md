#### Step 1: Extend SkillRegistry with dirty flag, mark_dirty(), and refresh()

- Modify: `src/tachikoma/skills/registry.py`
- What to do:
  - Store `_skills_path` as instance attribute in constructor (currently `skills_path` is a local var on line 63)
  - Update the existing `self._discover(skills_path)` call on line 64 to use `self._discover(self._skills_path)`
  - Add `_dirty: bool = False` instance attribute
  - Add `mark_dirty()` method that sets `_dirty = True`
  - Add `refresh()` method implementing swap-on-success. The mechanism works because `_discover()` writes directly to `self._agents` and `self._skills` instance attrs:
    1. If not dirty, return immediately (no-op)
    2. Save references to old `self._agents` and `self._skills`
    3. Assign fresh empty dicts to `self._agents` and `self._skills`
    4. Call `self._discover(self._skills_path)` — it populates the fresh dicts via `self._agents[...]` and `self._skills[...]`
    5. On success: set `_dirty = False`
    6. On exception: restore old dict references to `self._agents`/`self._skills`, log error, leave `_dirty = True` for retry

```python
# In SkillRegistry.__init__, replace the local var with instance attr:

self._skills_path = workspace_path / "skills"
self._dirty = False
self._discover(self._skills_path)
```

```python

    # External API for watcher — sets flag for next refresh() check
    ...

def refresh(self) -> None:
    # If not dirty, return (no-op)
    # Save old refs: old_agents, old_skills = self._agents, self._skills
    # Assign fresh dicts: self._agents, self._skills = {}, {}
    # Call self._discover(self._skills_path) — writes into the fresh dicts
    # On success: self._dirty = False
    # On exception: self._agents, self._skills = old_agents, old_skills; log error

```

- Test: `just test tests/skills/test_registry.py`

  - `refresh()` is no-op when `_dirty` is False (no filesystem access)
  - `refresh()` re-discovers skills from disk when dirty
  - New skills added to disk appear after `mark_dirty()` + `refresh()`
  - Deleted skills removed after `mark_dirty()` + `refresh()`
  - Swap-on-success: when `_discover()` raises, old dict references are restored
  - After failed refresh, `_dirty` remains True for retry

#### Step 2: Update bootstrap hook to create and store SkillRegistry

- Modify: `src/tachikoma/skills/hooks.py`
- What to do:
  - **Restructure the hook's control flow.** Currently the hook early-returns when the directory exists (lines 26-28), which would skip registry creation on non-first runs. The directory creation and registry creation must both happen unconditionally. Simplify by replacing the early-return with `mkdir(parents=True, exist_ok=True)` unconditionally, then create the registry.
  - Create `SkillRegistry(workspace_path)` and store in `ctx.extras["skill_registry"]`

```python
# In skills_hook — restructured flow:

async def skills_hook(ctx: BootstrapContext) -> None:
    workspace_path = ctx.settings_manager.settings.workspace.path
    skills_path = workspace_path / "skills"

    # Create directory if missing (unconditional, idempotent)
    skills_path.mkdir(parents=True, exist_ok=True)

    # Create shared registry (new — must happen on every launch)
    registry = SkillRegistry(workspace_path)
    ctx.extras["skill_registry"] = registry
```

  - Registry discovers skills that exist in the directory at hook time
  - Hook still creates directory when missing (existing behavior preserved)
  - Hook is idempotent (existing test preserved)
