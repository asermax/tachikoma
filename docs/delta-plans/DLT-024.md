#### Step 2: Add MIT LICENSE file

#### Step 3: Restructure CLI entry point

- Modify: `src/tachikoma/__main__.py`
- What to do:
  - Rename `@app.default async def main(channel)` to `@app.command async def run(channel)`
  - Add new `@app.default async def default_command()` that delegates via `await run()` â€” must be async since `run()` is async
  - Add `def cli(): app()` wrapper function at module level as the `[project.scripts]` entry point
  - Replace bare `app()` at bottom of file with `if __name__ == "__main__": cli()`
  - Update module docstring to reflect new structure

  ```python
  # In src/tachikoma/__main__.py, after app = App():

  # cli() wrapper for [project.scripts] entry point
  def cli():
      app()

  # ... existing imports and setup ...

  @app.command
  async def run(
      channel: Literal["repl", "telegram"] | None = None,
  ) -> None:
      """Run the Tachikoma agent."""
      # ... existing main() body unchanged ...

  @app.default
  async def default_command() -> None:
      await run()

  if __name__ == "__main__":
      cli()
  ```

- Test: `uv run tachikoma --help` shows `run` subcommand; `uv run tachikoma run --help` shows `--channel` flag; `uv run tachikoma` (bare) starts the agent with REPL default; `uv run tachikoma run --channel invalid` shows clear error

#### Step 4: Update justfile run recipe

- Modify: `justfile`
- What to do: Change `run` recipe from `PYTHONPATH=src uv run python -m tachikoma {{args}}` to `uv run tachikoma {{args}}`
- Test: `just run --help` works; `just` (bare invocation) still delegates to `run` correctly

#### Step 5: Add python-semantic-release configuration

- Modify: `pyproject.toml`
- What to do: Add `[tool.semantic_release]` section with:
  ```toml
  [tool.semantic_release]
  commit_parser = "conventional"
  version_toml = ["pyproject.toml:project.version"]
  build_command = "uv lock && uv build"
  major_on_zero = false
  tag_format = "v{version}"

  [tool.semantic_release.changelog]
  mode = "update"

  [tool.semantic_release.changelog.default_templates]
  changelog_file = "CHANGELOG.md"
  output_format = "md"
  ```
- Test: Config is valid TOML; PSR config keys match v10.x documentation

#### Step 6: Create GitHub Actions CD workflow

- Create: `.github/workflows/release.yml`
- What to do: Create two-job workflow:
  - Trigger: `push` to `master`
  - Concurrency: group `release`, `cancel-in-progress: false`
  - Default permissions: `contents: read`
  - **Job 1: `quality`** â€” checkout, `astral-sh/setup-uv`, `uv sync --all-groups`, run `ruff check .`, `ty check`, `pytest`
  - **Job 2: `release`** (needs: quality) â€” permissions: `contents: write`, `id-token: write`
    - Checkout with `fetch-depth: 0`
    - Setup uv, `uv sync --all-groups`
    - PSR action (`python-semantic-release/python-semantic-release@v10.x`) with `github_token`
    - Conditional on `steps.release.outputs.released == 'true'`:
      - `uv build`
      - `pypa/gh-action-pypi-publish` for PyPI OIDC upload
      - `python-semantic-release/publish-action@v10.x` to attach dist to GH release with `tag: ${{ steps.release.outputs.tag }}`
- Test: Workflow YAML is valid; job dependency chain `quality -> release` is correct; conditional gates reference correct output variables
