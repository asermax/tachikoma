- **Shinsenkyou `backend/app/agents/providers/claude/adapter.py`** — production reference for `ClaudeSDKClient` lifecycle (connect/disconnect), `receive_messages()` iteration patterns, tool result extraction (correlating ToolUseBlock with subsequent UserMessage ToolResultBlock), error handling (CLIConnectionError catch)
- `src/tachikoma/events.py` (from Batch 1) — domain types to produce

### Steps

#### Step 1: Create the message adapter

- Create: `src/tachikoma/adapter.py`
- What to do: Implement `adapt(sdk_message) -> list[AgentEvent]` as a pure function following the design's mapping table exactly. Handle each SDK type:
  - `AssistantMessage`: check `.error` field first → `Error` event (rate_limit/server_error → `recoverable=True`, auth/billing → `recoverable=False`). Then iterate `.content` blocks — `TextBlock` → `TextChunk`, `ToolUseBlock` → `ToolActivity` (extract tool name, input params; note: tool result comes in subsequent `UserMessage` as `ToolResultBlock` — for v1 the result field can be left empty or populated if available on the block).
  - `ResultMessage`: `is_error=False` → `Result` (extract session_id, cost, usage), `is_error=True` → `Error`.
  - `UserMessage` → empty list (filtered).
  - `SystemMessage` → empty list (filtered).

  This is the only module that imports SDK message types.
- Test: `just typecheck` passes

#### Step 3: Create the coordinator

- Create: `src/tachikoma/coordinator.py`
- What to do: Implement the `Coordinator` class:
  - **Constructor**: takes allowed_tools list and optional model string. Stores config, no client yet.
  - **Async context manager**: `__aenter__` creates `ClaudeSDKClient` with `ClaudeAgentOptions` and calls `client.connect()`. `__aexit__` calls `client.disconnect()`.
  - **`send_message(text) -> AsyncIterator[AgentEvent]`**: calls `client.query(text)`, iterates `client.receive_messages()`, runs each message through `adapt()`, yields resulting events. Breaks on `ResultMessage`.
  - **Error boundary**: wraps the `receive_messages()` iteration in try/except for `CLIConnectionError` and `ProcessError` — catches and yields `Error(recoverable=True)` so the conversation remains usable after transient failures.
  - **`interrupt()`**: delegates to `client.interrupt()`.
- Test: `just typecheck` passes

## Batch 3: Terminal REPL, Entry Point & Verification — ⧗ Pending
Depends on: Batch 2

### Context & Research
- **Spec R3** — all REPL acceptance criteria (streaming, context, Ctrl+C/D, empty input, mid-stream error)
- **Spec R0 AC** — API key missing → fail fast with clear error
- **Design S5** — prompt_toolkit usage: PromptSession, FileHistory at `~/.tachikoma/repl_history`, Validator, event rendering (TextChunk → inline print, ToolActivity → dimmed status, Result → newline, Error → message), Ctrl+C/D handling
- **Design S6** — error boundary: SDK startup exceptions (CLINotFoundError, CLIConnectionError, ProcessError) for `__main__.py` fast-fail; Error event rendering in REPL (recoverable → continue, non-recoverable → exit)
- **Design "System Behavior" scenarios**: Normal conversation turn, Ctrl+C during streaming, Ctrl+C at prompt, Ctrl+D at empty prompt, Empty input prevented, Authentication failure on startup, In-stream error, Transient connection error mid-stream, Agent uses a tool
- **Design "Data Flow"** — Startup flow (steps 1-8), Shutdown flow (steps 1-6)
- **prompt_toolkit docs** — `PromptSession.prompt_async()`, `FileHistory` (auto-creates file, needs existing parent directory), `Validator.from_callable()`, `KeyboardInterrupt` on Ctrl+C, `EOFError` on Ctrl+D with empty buffer
- `src/tachikoma/coordinator.py` (from Batch 2) — Coordinator API: async context manager, send_message(), interrupt()
- `src/tachikoma/events.py` (from Batch 1) — AgentEvent types to render

### Steps

#### Step 1: Create the REPL module

- Create: `src/tachikoma/repl.py`
- What to do: Implement the `Repl` class taking a `Coordinator` reference. Creates a `PromptSession` with `FileHistory` at `~/.tachikoma/repl_history` and a `Validator` that rejects empty/whitespace-only input. The `run()` async method is the main loop:
  - `prompt_async()` for input — catch `KeyboardInterrupt` (Ctrl+C) → exit, catch `EOFError` (Ctrl+D) → exit
  - `coordinator.send_message(text)` for response — iterate events and render each
  - During streaming, catch `KeyboardInterrupt` → call `coordinator.interrupt()`, then exit
  - Rendering: `TextChunk` → `print(text, end="", flush=True)`, `ToolActivity` → dimmed status line with tool name and key params (e.g., "Reading src/main.py..."), `Result` → newline, `Error` → print error message; if `not recoverable` → break out of REPL loop
- Test: `just typecheck` passes

Wrap coordinator context entry in try/except for SDK startup errors (`CLINotFoundError`, `CLIConnectionError`, `ProcessError`) → print to stderr, `sys.exit(1)`.
