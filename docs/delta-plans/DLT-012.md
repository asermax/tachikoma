#### Step 2: Create the Settings model hierarchy with tests

- Create: `src/tachikoma/config.py`, `tests/test_config.py`
- What to do: Define the typed settings model with nested sections. The module should contain:
  - `CONFIG_PATH` constant pointing to `~/.config/tachikoma/config.toml`
  - `WorkspaceSettings` with `path: Path` defaulting to `~/tachikoma`, with a `field_validator` that calls `Path.expanduser()`
  - `AgentSettings` with `model: str | None = None` and `allowed_tools: list[str]` defaulting to `["Read", "Glob", "Grep"]`
  - `Settings` root model with `workspace: WorkspaceSettings` and `agent: AgentSettings`, defaulting to their respective defaults
  - All models use `ConfigDict(frozen=True, extra="ignore")`
- Test: Create `tests/test_config.py` with `TestSettingsModel` class covering: model construction with defaults, path `~` expansion, frozen immutability (mutation raises), and `extra="ignore"` behavior. Run `uv run pytest tests/test_config.py`

```python

# WorkspaceSettings with path validator for ~ expansion
# AgentSettings with model and allowed_tools

```

#### Step 3: Create the default config generator with tests

- Modify: `src/tachikoma/config.py`, `tests/test_config.py`
- What to do: Add a `_generate_default_config()` private function that:
  - Creates the `~/.config/tachikoma/` directory (with `parents=True`)
  - Uses `tomlkit` to programmatically build a TOML document where all key-value pairs are commented out and annotated with descriptions from model field metadata
  - Writes the document to `CONFIG_PATH`
  - The generated file should parse to an empty dict (all values are comments), so all defaults apply
- Test: Add `TestDefaultConfigGeneration` class to `tests/test_config.py`. Use `tmp_path` + monkeypatch to redirect `CONFIG_PATH`. Tests: generated file parses to empty dict, file contains comments for all fields, directory creation works, permission error on directory creation raises `SystemExit`. Run `uv run pytest tests/test_config.py`

```python

# _generate_default_config(config_path: Path) function
# - Accept path parameter for testability (defaults to CONFIG_PATH)
# - Build tomlkit document with commented sections for [workspace] and [agent]
# - Each field gets a comment with its description and a commented-out key=value line

```

#### Step 4: Create the config loader function with tests

- Modify: `src/tachikoma/config.py`, `tests/test_config.py`
- What to do: Add `load_settings()` function following the design's data flow:
  1. Check if config file exists at `CONFIG_PATH`
  2. If not: call `_generate_default_config()` to create directory and default file
  3. If exists but not a regular file: exit with clear error
  4. If exists but not readable: exit with clear error
  5. Read TOML via `tomllib.load()`
  6. If invalid TOML: exit with parse error
  7. Validate via `Settings.model_validate(data)`
  8. If validation fails: format Pydantic's `ValidationError` into user-friendly message (field path, expected type, actual value)
  9. Return frozen `Settings` instance
  - Use `SystemExit` with formatted message for all error exits
- Test: Add `TestLoadSettings` class to `tests/test_config.py`. Use `tmp_path` + monkeypatch. Tests covering all spec ACs: no file (auto-generates + loads defaults), empty file (all defaults), partial config (specified values + defaults), invalid type (SystemExit with field name), invalid TOML (SystemExit with parse error), permission denied (SystemExit), path-is-directory (SystemExit), unknown keys silently ignored. Run `uv run pytest tests/test_config.py`

```python

# load_settings(config_path: Path | None = None) -> Settings
# - Accept optional path for testability (defaults to CONFIG_PATH)
# - Follow the flowchart from the design document
# - Format Pydantic ValidationError using loc, msg, and input fields
# - Use SystemExit for all error exits (clear messages)
```

- What to do:

  - Call `load_settings()` as the first operation in `main()`
