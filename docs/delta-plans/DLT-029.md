### Context & Research
- `docs/delta-specs/DLT-029.md` (R2, R3, R3.1, R3.2, R5 ACs) — read tool removal, remove tool single/batch/error semantics, tool descriptions
- `docs/delta-designs/DLT-029.md` (S2, S3, S5) — tool removal, snapshot closure, all-or-nothing validation, file rewrite from immutable snapshot, file deletion when empty
- `src/tachikoma/context/tools.py` — current `read_pending_signals` and `add_pending_signal` implementations, `PENDING_SIGNALS_HEADER` constant
- `claude_agent_sdk` `tool()` API — the simple schema form (`{"signal": str}`) only supports base types; `list[int]` requires full JSON Schema: `{"type": "object", "properties": {"indices": {"type": "array", "items": {"type": "integer"}}}, "required": ["indices"]}`

### Steps

#### Step 6: Remove read_pending_signals and add remove_pending_signal tool

- Modify: `src/tachikoma/context/tools.py`
- What to do:

  2. Remove the entire `read_pending_signals` `@tool` definition
  3. Add `remove_pending_signal` tool using `@tool()` with full JSON Schema for `{"indices": list[int]}`. The tool: validates indices (empty = no-op success), converts 1-based to 0-based, validates all indices in range (all-or-nothing), computes filtered entries from immutable snapshot, rewrites file (or deletes if empty)
