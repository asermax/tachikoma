#### Step 12: Replace coordinator's in-memory context management with DB-driven assembly

- Modify: `src/tachikoma/coordinator.py`
- What to do: This is the core atomic change. Replace the coordinator's in-memory context state with DB-driven assembly in a single pass:
  1. **Constructor**: Replace `system_prompt: str | None` param with `foundational_context: str | None`. Store as `self._foundational_context`. Remove `self._base_system_prompt`, `self._previous_summary`, and `self._bridging_context` instance variables.
  2. **`_build_options()`**: Simplify to accept `system_prompt_append: str | None` param. Remove all inline `_previous_summary`/`_bridging_context` assembly logic. Just set `SystemPromptPreset(append=system_prompt_append)` when provided.
  3. **`send_message()`**: After session creation, save foundational entry via registry. After pre-processing, save provider entries. Before `_build_options()`, load entries from DB, call `build_system_prompt(entries)`, pass result to `_build_options()`. On load failure, log error and fall back to `build_system_prompt([])` (SYSTEM_PREAMBLE only).
  4. **Remove `assemble_context` usage**: Remove the import and call — pre-processing results are now saved to DB and assembled into the system prompt, not prepended to message text.
  5. **Add imports**: `build_system_prompt` from `context.assembly`, `SessionContextEntry` from `sessions.model`.
