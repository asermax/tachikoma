#### Step 3: Rewrite cron evaluation and firing logic

- Modify: `src/tachikoma/tasks/scheduler.py`
- What to do: Replace the cron evaluation block (lines 84‚Äì115) and restructure the duplicate check:
  1. **Compute anchor**: If `last_fired_at` is None, use start-of-hour in `tz`. If set, convert from UTC to `tz` via `.astimezone(tz)`.
  2. **Get next fire time**: `next_fire = next(CronSim(expression, anchor_tz))`
  3. **Strict firing condition**: `should_fire = next_fire <= now_tz` (no tolerance)
  4. **Compute cron match UTC**: `cron_match_utc = next_fire.astimezone(UTC)`
  5. **Set `scheduled_for`**: Use `cron_match_utc` instead of `now_utc`
  6. **Restructure duplicate check**: The current code has a single `get_active_instance_for_definition` call (line 131) shared by both cron and one-shot paths. Move this into each branch: cron path passes `scheduled_for=cron_match_utc`, one-shot path calls without `scheduled_for` (preserving current behavior for R5)


#### Step 4: Add safety net logging, fast-forward, and update `last_fired_at`

- Modify: `src/tachikoma/tasks/scheduler.py`
- What to do:
  1. **S3 logging**: When the safety net (duplicate check in cron path) finds an existing instance, emit an info-level log: `_log.info("Duplicate suppressed for {name} ‚Äî period {match} already covered", name=definition.name, match=cron_match_utc)`
  2. **Fast-forward `last_fired_at`**: After creating the cron instance, iterate CronSim forward from `next_fire` (in tz) to find the latest match at or before `now_tz`, convert to UTC. This ensures catch-up creates only one instance per restart.
  3. **Update definition**: Use fast-forwarded time for `last_fired_at` instead of `now_utc`

```python
# After creating the cron instance, fast-forward last_fired_at:

# Advance to latest cron match <= now_tz to skip missed periods
last_match = next_fire
for future_match in CronSim(schedule.expression, next_fire):
    if future_match > now_tz:
        break
    last_match = future_match

last_fired_utc = last_match.astimezone(UTC)

await repository.update_definition(
    definition.id,
    last_fired_at=last_fired_utc,
)
```
