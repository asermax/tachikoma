#### Step 7: Migrate existing TestRendering tests

- Modify: `tests/test_repl.py`
- What to do: Replace all `capsys` parameters with `stdout_buf`/`stderr_buf` fixtures. Change assertions from `capsys.readouterr().out` to `stdout_buf.getvalue()` and `.err` to `stderr_buf.getvalue()`. Use content-presence assertions (e.g., `assert "Hello" in stdout_buf.getvalue()`) since rich adds formatting around text. Specific test changes:
  - **Adapt** `test_renders_text_chunk_inline` — assert `"Hello"` in stdout output
  - **Adapt** all tool activity tests — assert label text in stdout output (same pattern)
  - **Adapt** `test_tool_activity_after_text_gets_leading_newline` — assert both `"thinking"` and `"Reading f.py..."` appear in stdout output (preserves observable behavior that both events produce output, even though `_needs_newline` mechanism is gone)
  - **Remove** `test_tool_activity_without_prior_text_has_no_leading_newline` — this tested `_needs_newline` internal behavior that no longer exists with block rendering
  - **Adapt** `test_renders_result_as_newline` — assert stdout output contains a newline
  - **Adapt** `test_renders_error_message` — assert `"connection lost"` in stderr output
  - **Keep unchanged** return value tests (recoverable/non-recoverable) — they don't use capsys

- Test: `uv run pytest tests/test_repl.py::TestRendering` — all tests pass

#### Step 8: Add markdown rendering test cases

- Modify: `tests/test_repl.py`
- What to do: Add new test cases to `TestRendering` covering the markdown-specific acceptance criteria:
  - Markdown heading renders with content present and formatting (e.g., render `"# Title"`, assert `"Title"` in output)
  - Plain text renders without markdown artifacts (render `"Hello world"`, assert `"Hello world"` in output)
  - Fenced code block content is preserved in output (render a code block, assert the code appears)
  - Sequential TextChunks each produce output (render two chunks, assert both contents present)
