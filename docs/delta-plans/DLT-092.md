### Steps

#### Step 1: Add `_detect_system_timezone()` helper to `config.py`

- Modify: `src/tachikoma/config.py`
- What to do: Add a module-level `_detect_system_timezone() -> str` function that resolves `/etc/localtime` symlink target, extracts the IANA timezone name (everything after `zoneinfo/` in the path), and returns it. Falls back to `"UTC"` if `/etc/localtime` doesn't exist, isn't a symlink, or the path doesn't contain `zoneinfo/`.
- Test: Unit test `_detect_system_timezone()` — mock `Path.resolve()` to simulate symlink targets; test fallback to "UTC" when file doesn't exist or path format is unexpected.

#### Step 2: Add Pydantic validators on `TaskSettings.timezone`

- Modify: `src/tachikoma/config.py`
- What to do: Add two `field_validator`s on `TaskSettings.timezone`:
  1. `mode='before'`: if value is `""`, call `_detect_system_timezone()` and return the result
  2. `mode='after'`: call `ZoneInfo(v)` to validate; catch `KeyError` (parent of `ZoneInfoNotFoundError`) and raise `ValueError` with a clear message like `"'{v}' is not a valid IANA timezone"`
  - Import `ZoneInfo` from `zoneinfo` at module top
  - Keep `default=""` on the field for config generation compatibility
- Test: `TaskSettings()` resolves to non-empty tz; `TaskSettings(timezone="US/Eastern")` preserves; `TaskSettings(timezone="Fake/Timezone")` raises `ValidationError`.
