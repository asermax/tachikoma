### Steps

#### Step 1: Create post-processing pipeline module

- Create: `src/tachikoma/post_processing.py`
- What to do:
  - Define `PostProcessor` ABC with a single abstract method `async def process(session: Session) -> None`
  - Define `PostProcessingPipeline` class:
    - `_processors: list[PostProcessor]` and `_lock: asyncio.Lock` in `__init__`
    - `register(processor: PostProcessor)` appends to the list
    - `async run(session: Session)` acquires the lock, runs `asyncio.gather(*[p.process(session) for p in self._processors], return_exceptions=True)`, iterates results and logs any `Exception` instances per DES-002
  - Define standalone `async fork_and_consume(session: Session, prompt: str, cwd: Path) -> None`:
    - Creates `ClaudeAgentOptions(cwd=cwd, resume=session.sdk_session_id, fork_session=True)`
    - Calls `query(prompt=prompt, options=options)`
    - Fully consumes the returned async iterator (`async for _ in result: pass`)
  - Module-level bound logger: `_log = logger.bind(component="post_processing")`

```python
# src/tachikoma/post_processing.py

# PostProcessor ABC — interface only, no SDK coupling
# PostProcessingPipeline — generic parallel runner with error isolation
# fork_and_consume — standalone helper for SDK session forking

# NOTE: Design S3 shows resume/fork_session as query() params — the actual SDK
# signature puts them on ClaudeAgentOptions. Implementation follows the verified
# SDK signature: ClaudeAgentOptions(cwd=cwd, resume=..., fork_session=True)
# then query(prompt=prompt, options=options)
```

- Verify: module imports successfully (`python -c "from tachikoma.post_processing import PostProcessor, PostProcessingPipeline, fork_and_consume"`)

#### Step 2: Create tests for post-processing pipeline

- Create: `tests/test_post_processing.py`

  - `TestPostProcessingPipeline`:
    - `test_runs_all_registered_processors` — register 2+ mock processors, verify all `process()` awaited with same session
    - `test_error_isolation_continues_other_processors` — one processor raises, verify the other completes normally
    - `test_logs_processor_failures` — verify exception is logged per DES-002
    - `test_returns_after_all_complete` — verify pipeline awaits all processors before returning
    - `test_serializes_concurrent_invocations` — two concurrent `run()` calls execute sequentially (lock test)
    - `test_runs_with_no_registered_processors` — empty pipeline runs without error
  - `TestForkAndConsume`:

    - `test_consumes_full_async_iterator` — mock `query()` returning async iterator, verify fully consumed
    - `test_propagates_query_error` — mock `query()` raising, verify exception propagates
  - Use `mocker.patch()` for `query()`, follow DES-001 patterns

#### Step 5: Create EpisodicProcessor

- Create: `src/tachikoma/memory/episodic.py`
- Modify: `src/tachikoma/memory/__init__.py` — add `EpisodicProcessor` to re-exports
- What to do:
  - Define `EPISODIC_PROMPT` module-level constant — extraction prompt that instructs the agent to:
    - Read existing files in `memories/episodic/`
    - Analyze the conversation for meaningful events
    - Create or update date-stamped files (`YYYY-MM-DD.md`) with conversation summaries
    - Consolidate same-day entries rather than creating duplicates
    - Creating nothing is acceptable if the conversation has no meaningful content
    - Only modify files within `memories/episodic/`
  - Define `EpisodicProcessor(PostProcessor)`:
    - `__init__(self, cwd: Path)` stores `_cwd`
    - `async process(self, session: Session)` calls `fork_and_consume(session, EPISODIC_PROMPT, self._cwd)`

```python
# src/tachikoma/memory/episodic.py

# EpisodicProcessor — thin wrapper around fork_and_consume
# EPISODIC_PROMPT — instructs the forked agent:
#   1. Read memories/episodic/ to see existing summaries
#   2. Analyze the conversation for meaningful events
#   3. Create/update YYYY-MM-DD.md with day's summary
#   4. Consolidate if same-day file exists
#   5. No changes is valid for trivial conversations
```

#### Step 9: Modify Coordinator to accept and trigger pipeline

- Modify: `src/tachikoma/coordinator.py`
- What to do:

  - Add optional `pipeline: PostProcessingPipeline | None = None` parameter to `__init__`
  - Store as `self._pipeline`
  - Modify `__aexit__` — the critical ordering is:
    1. Capture active session reference via `get_active_session()` **BEFORE** `close_session()` (the registry clears `_active_session` on close — this capture is already done in current code with `active = await self._registry.get_active_session()`)
    2. Close session via `close_session(active.id)` (already done)
    3. **NEW**: Use the captured `active` reference to conditionally trigger the pipeline:
       - If `active.sdk_session_id is not None` AND `self._pipeline is not None`: `await self._pipeline.run(active)` wrapped in try/except
       - If `active is not None` and `active.sdk_session_id is None`: log warning per DES-002
       - On exception: log error per DES-002, continue to disconnect
    4. Disconnect SDK client (already done)
