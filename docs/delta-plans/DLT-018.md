#### Step 10: Implement `CoreContextProcessor` with prompt

- Create: `src/tachikoma/context/processor.py`
- What to do: Implement the processor and its prompt constant in the same module:

```python
# In src/tachikoma/context/processor.py:

# CONTEXT_UPDATE_PROMPT (module-level constant) instructs the forked agent to:
# 1. Read all three context files (context/SOUL.md, USER.md, AGENTS.md)
# 2. Read pending signals via the read_pending_signals tool
# 3. Analyze the completed conversation for context-relevant information
# 4. Classify each signal:
#    - Clear & explicit → update the appropriate context file directly
#      (read file first, preserve structure, merge contextually)
#    - Ambiguous / one-off → check pending signals for semantic recurrence
#      → If recurring: promote to context file update
#      → If new: stage via add_pending_signal tool
#    - No relevant info → do nothing
# 5. Constraints in the prompt:
#    - Conservative: only high-confidence changes
#    - Route to correct file (personality→SOUL, user info→USER, instructions→AGENTS)
#    - Read-first: always read file before modifying
#    - Only interact with pending signals through provided tools
#    - Don't modify or delete pending signals directly

# CoreContextProcessor(PromptDrivenProcessor):
#   __init__(self, cwd: Path):
#     - Compute _data_dir = cwd / ".tachikoma"
#     - Call super().__init__(CONTEXT_UPDATE_PROMPT, cwd)
#
#   async def process(self, session: Session) -> None:
#     NOTE: Full override — does NOT call super().process()
#     1. Pre-step: call clean_pending_signals(self._data_dir)
#     2. Create MCP server: create_pending_signals_server(self._data_dir)
#     3. Snapshot context file mtimes:
#        - For each (filename, _, _) in CONTEXT_FILES:
#          - stat cwd / CONTEXT_DIR_NAME / filename → record mtime (or None if missing)
#     4. Fork: await fork_and_consume(session, self._prompt, self._cwd,
#              mcp_servers={"pending-signals": server})
#     5. Post-step: compare mtimes, log changed files per DES-002
```
