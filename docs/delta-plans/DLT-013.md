#### Step 4: Create logging.py module

- Create: `src/tachikoma/logging.py`
- What to do:
  1. Define the structured text format constant: `LOG_FORMAT = "{time:YYYY-MM-DD HH:mm:ss.SSS} | {level: <8} | {extra[component]} | {name}:{function}:{line} - {message}"`
  2. Implement `configure_logging(level: str, data_path: Path, console: bool = False) -> None`:
     - `logger.configure(extra={"component": "core"})` — set global default so format string works even for unbound loggers
     - `logger.remove()` — strip default stderr handler (ensures idempotency on repeated calls)
     - `logger.add()` for file handler at `data_path / "logs" / "tachikoma.log"` with: `format=LOG_FORMAT`, `level=level`, `rotation="100 MB"`, `retention="7 days"`, `compression="gz"`, `enqueue=True`, `encoding="utf-8"`, `diagnose=True`
     - If `console=True`, `logger.add(sys.stderr, format=LOG_FORMAT, level=level, colorize=True, diagnose=True)`
  3. Implement `InterceptHandler(logging.Handler)`:
     - `emit(self, record)` maps stdlib record level to loguru level name
     - Uses frame-walking to find correct depth, then `logger.opt(depth=depth, exception=record.exc_info).log(level, record.getMessage())`
  4. At end of `configure_logging()`, install interception: `logging.basicConfig(handlers=[InterceptHandler()], level=0, force=True)`
  5. Implement `async def logging_hook(ctx: BootstrapContext) -> None` (**must be async** — `BootstrapHook` type requires `Awaitable`):
     - Read settings from `ctx.settings_manager.settings`
     - Compute `logs_dir = settings.workspace.data_path / "logs"`
     - Create `logs_dir` with `exist_ok=True`; on `PermissionError`, raise `RuntimeError(f"Cannot create logs directory: Permission denied: {logs_dir}")` — note: `Bootstrap.run()` wraps this in `BootstrapError(f"Hook 'logging' failed: ...")`, satisfying the AC requirement to name the hook
     - Call `configure_logging(settings.logging.level, settings.workspace.data_path, settings.logging.console)`
- Test: Unit tests in next step
