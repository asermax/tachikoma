# Implementation Plan: DLT-023 — Bootstrap agent workspace on first run

**Delta Spec**: [../delta-specs/DLT-023.md](../delta-specs/DLT-023.md)
**Delta Design**: [../delta-designs/DLT-023.md](../delta-designs/DLT-023.md)

## Context

The agent currently has a single ad-hoc `settings.workspace.path.mkdir(parents=True, exist_ok=True)` in `__main__.py`. As more modules need first-run setup (git init, core context files, skills directory), this approach doesn't scale. DLT-023 introduces a formal bootstrap system with named hooks, a read-write `SettingsManager`, and sets the agent's working directory via the SDK's `cwd` option.

#### Step 2: Add SettingsManager class

- Modify: `src/tachikoma/config.py`
- What to do: Add `SettingsManager` class that wraps config loading with read-write access. Constructor accepts optional `config_path` (defaults to `CONFIG_PATH`), calls `load_settings()` internally, and loads the tomlkit document from the file for later write operations.

```python
# After load_settings() function:

class SettingsManager:
    # __init__(config_path=None): load Settings + tomlkit doc
    # .settings property: returns frozen Settings snapshot
    # .update(section, key, value): modify in-memory tomlkit doc; KeyError on invalid section/key
    # .save(): write tomlkit doc to file, reload frozen Settings via load_settings()
```

Key implementation details:
- `update()` validates section against `Settings.model_fields` and key against the section model's `model_fields`, raising `KeyError` on invalid section/key
- `update()` creates the TOML section table if it doesn't exist yet
- `save()` writes the tomlkit document, then reloads `_settings` via `load_settings()` so the frozen snapshot is current
- The tomlkit document is loaded once in `__init__` and kept in memory for subsequent updates

```python
#   settings_manager: SettingsManager

# BootstrapHook = Callable[[BootstrapContext], None]

#   __init__(settings_manager, prompt=input)
#     - Creates BootstrapContext internally
#     - Initializes empty hook list
#   register(name, hook) — adds (name, hook) tuple to ordered list
#   run() — executes hooks in registration order
#     - Catches exceptions, wraps in BootstrapError naming the hook
#     - Uses `raise BootstrapError(...) from exc` for chaining
```

```python
# workspace_hook(ctx: BootstrapContext) -> None

#      - exists and is dir → continue (don't return — still need to check data_path)
```
