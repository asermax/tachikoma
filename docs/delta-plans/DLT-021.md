#### Step 7: Implement LLM classification and response parsing

- Modify: `src/tachikoma/skills/context_provider.py`
- What to do: In `provide()`, after the empty check:
  1. Format skills list from `self._registry.skills` (name + description per skill)
  2. Build prompt with `SKILL_CLASSIFICATION_PROMPT.format(skills=..., message=...)`
  3. Create `ClaudeAgentOptions(model="opus", effort="low", max_turns=3, permission_mode="bypassPermissions", cwd=self._cwd, cli_path=self._cli_path)` — no `allowed_tools` (pure reasoning, no file access needed)
  4. Call `query(prompt=prompt, options=options)` and fully consume the async generator per DES-005
  5. On `ResultMessage`: if `is_error` → log warning, set result to None; if result is `NO_RELEVANT_SKILLS` → return None; otherwise parse as skill names (one per line, stripped)
  6. Filter parsed names against `self._registry.skills` keys (discard unrecognized)
  7. Wrap entire flow in try/except → log per DES-002, return None on failure

  Note: `max_turns=3` follows the design. Can be tightened to `1` if testing confirms single-turn behavior.
- Test: Mock `query()` — verify correct options (including `cli_path`), sentinel handling, name filtering, error handling

#### Step 8: Implement SKILL.md body reading and XML assembly

- Modify: `src/tachikoma/skills/context_provider.py`
- What to do: After filtering valid skill names:
  1. For each detected skill name, read `self._cwd / "skills" / name / "SKILL.md"` using `frontmatter.load()`, take `.content` (body without YAML). Note: this path construction duplicates `SkillRegistry._load_skill()` — this is a conscious design choice ("SKILL.md body read at detection time").
  2. Record skill directory path: `self._cwd / "skills" / name`
  3. If read fails for a skill → log warning, skip that skill (graceful degradation, other skills proceed)
  4. Assemble `<skills>` XML block with each skill clearly delineated
  5. Filter `self._registry.get_agents()` into a **new** dict (don't mutate registry's internal dict): keep entries whose key starts with `{detected_skill_name}/`
  6. Return `ContextResult(tag="skills", content=xml_block, agents=filtered_agents or None)`
  7. If no valid skills remain after reading → return None

```python
# XML block format per skill:
# <skill name="skill-name" directory="path/to/skills/skill-name">
# SKILL.md body content here
# </skill>
```

- Test: Test with filesystem-based skills, verify XML format, agent filtering, partial failure handling
