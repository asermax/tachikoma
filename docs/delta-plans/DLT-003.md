#### Step 5: Create SkillRegistry class with skill discovery and SKILL.md parsing

- Create: `src/tachikoma/skills/registry.py`
- What to do:
  - Define a `Skill` dataclass to hold skill metadata: `name: str`, `description: str`, `version: str | None`
  - Define `SkillRegistry` class with constructor accepting `workspace_path: Path`
  - In `__init__`, call internal discovery method that:
    1. Resolves `workspace_path / "skills"` — if it doesn't exist, set agents dict to empty and return
    2. Iterates all subdirectories (ignoring regular files)
    3. For each directory, calls `_load_skill(skill_dir)`
  - `_load_skill` method:
    1. Checks for `SKILL.md` — warns and skips if missing
    2. Parses YAML frontmatter using `frontmatter.load()`
    3. Validates: name present and non-empty, description present and non-empty, name matches folder name
    4. Creates Skill metadata object, stores in `_skills` dict
    5. Calls `_load_agents(skill_dir, skill_name)` for agent discovery
  - Expose `get_agents() -> dict[str, AgentDefinition]` method
  - Expose `skills` property returning `dict[str, Skill]` for metadata access (minimal — just the data structure and property, for DLT-021)
  - Module-level bound logger: `_log = logger.bind(component="skills")`
  - All exceptions caught per-skill and logged as warnings (never propagated)


#### Step 6: Add agent definition parsing and AgentDefinition construction

- Modify: `src/tachikoma/skills/registry.py`
- What to do:
  - Add `_load_agents(skill_dir, skill_name)` method:
    1. Check if `skill_dir / "agents"` exists — if not, return (valid: skill without agents)
    2. Iterate all `.md` files in agents/ directory (ignore non-.md files)
    3. For each `.md` file, call `_load_agent(agent_path, skill_name)`
  - Add `_load_agent(agent_path, skill_name)` method:
    1. Parse with `frontmatter.load(agent_path)`
    2. Extract: description (required), model (optional — passthrough, no registry-side validation per design decision), tools (optional list)
    3. Agent name = filename stem (e.g., `extractor.md` → `extractor`)
    4. Validate: description present and non-empty
    5. Construct `AgentDefinition(description=..., prompt=post.content, model=..., tools=...)`
    6. Add to `_agents` dict with namespace key `skill_name/agent_name`
    7. All exceptions caught and logged as warnings
