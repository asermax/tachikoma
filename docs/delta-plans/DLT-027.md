#### Step 2: Migrate BootstrapContext to include extras field

- Modify: `src/tachikoma/bootstrap.py`
- What to do: Add `extras: dict[str, Any]` field to the `BootstrapContext` frozen dataclass with `default_factory=dict`. Add the `Any` import from `typing` and `field` from `dataclasses`. The dataclass remains `frozen=True` -- Python's frozenness prevents reassigning the field reference (`ctx.extras = {}` would raise `FrozenInstanceError`), but the mutable dict contents can be modified freely (`ctx.extras["key"] = value` works). This is the intended design: hooks write into the dict, but the dict itself cannot be swapped out.

```python
# In src/tachikoma/bootstrap.py, update BootstrapContext:

@dataclass(frozen=True)
class BootstrapContext:
    settings_manager: SettingsManager
    prompt: Callable[[str], str]
    extras: dict[str, Any] = field(default_factory=dict)
    # frozen=True prevents ctx.extras = {} but allows ctx.extras["key"] = value
```

- Test: Existing tests still pass (extras defaults to empty dict, no existing code uses it)

#### Step 3: Migrate BootstrapHook type, Bootstrap.run(), and workspace_hook to async

- Modify: `src/tachikoma/bootstrap.py`
- What to do: Change `BootstrapHook` type alias from `Callable[[BootstrapContext], None]` to `Callable[[BootstrapContext], Awaitable[None]]`. Add `Awaitable` to the imports from `collections.abc`. Change `Bootstrap.run()` from `def run(self)` to `async def run(self)` and change `hook(self._context)` to `await hook(self._context)` inside the loop. Change `workspace_hook` from `def workspace_hook(ctx)` to `async def workspace_hook(ctx)` -- the body is unchanged since filesystem operations are sync, only the signature changes to satisfy the new `BootstrapHook` type.

```python
# In src/tachikoma/bootstrap.py:

from collections.abc import Awaitable, Callable

BootstrapHook = Callable[[BootstrapContext], Awaitable[None]]

# In Bootstrap class:
async def run(self) -> None:
    for name, hook in self._hooks:
        try:
            await hook(self._context)
        except Exception as exc:
            raise BootstrapError(f"Hook '{name}' failed: {exc}") from exc

# workspace_hook gains async keyword, body unchanged:
async def workspace_hook(ctx: BootstrapContext) -> None:
    ...
```

- Test: Import succeeds. Note: existing tests will break at this point because they use sync hooks and sync `run()` -- this is expected and fixed in Step 5.

## Notes

> **Batch status markers**: Pending -> Implementing -> Done
> Updated by the implementing agent as it progresses through batches.

> **Context & Research entries**: Each entry explains *why* it's needed. Mix of project docs
> (specs, designs, ADRs, DES), code files, and external research pointers (library docs, patterns
> to investigate). These are a starting point -- the implementing agent adapts them as it goes
> (removing irrelevant ones, adding discovered files, updating references).

> **Code snippets in steps**: Use snippets to show *where* code goes and *what* it should do
> (via comments), not the complete implementation. The implementation agent has full access to
> spec, design, and decision documents -- plans provide structural guidance, not code.
